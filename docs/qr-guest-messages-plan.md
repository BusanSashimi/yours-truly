# Implementation plan — QR guest messages (비공개 축하 메시지·사진)

**Status:** approved — Phase 1 ready to implement · **Author:** Claude · **Date:** 2026-06-13 (updated 2026-06-14)

## 1. Goal

A wedding guest scans a QR code (printed on table cards / signage, generated by
the couple in the dashboard) → lands on a focused mobile page → leaves a
**message and/or photos** → it is delivered **privately to the couple** as a
single unified post in their dashboard inbox. Nothing is shown publicly.

This is distinct from the existing public **방명록** (guestbook) and
**게스트스냅** (public photo wall): those are public, listed on the invitation
page; this is a **private inbox** keyed off a QR-reachable submit page.

### Locked decisions

| Decision | Choice |
|---|---|
| Destination of submissions | **Private couple inbox** (new table; dashboard-only) |
| What the QR opens | **Dedicated submit page** `/invitations/<slug>/send` |
| Submission shape | **Unified post**: one message + 0–N attached photos |
| QR generation | **Client-side**, `qrcode.react` (SVG + PNG/SVG download for print) |
| Photo privacy | **Truly private** — photos under a non-public `m/` prefix; owner inbox views them via short-lived presigned GET. `m/*` is **not** added to the public-read policy. (Public `g/` model rejected: weaker privacy.) |

### Default decisions (flip before Phase 1 if desired)

- **`message` not required** on its own — a post needs **at least one** of
  message or ≥1 photo.
- **Photo cap = 10 per message**; **15 MB** per photo (reuses
  `GUEST_UPLOAD_MAX_BYTES`); types jpeg/png/webp.
- **Read/unread tracking** via a nullable `readAt` (cheap, enables an inbox
  unread badge).

## 3. Architecture

### 3.1 Data model — new table `guest_messages`

`apps/api/src/db/schema.ts` (mirrors `guest_uploads` style at `schema.ts:174`):

```ts
export const guestMessages = pgTable(
  "guest_messages",
  {
    id: uuid("id").primaryKey().defaultRandom(),
    invitationId: uuid("invitation_id")
      .notNull()
      .references(() => invitations.id, { onDelete: "cascade" }),
    senderName: text("sender_name"),               // optional, ≤40
    message: text("message"),                       // optional, ≤1000
    photoKeys: text("photo_keys").array().notNull().default([]), // m/<id>/<uuid>.<ext>
    readAt: timestamp("read_at", { withTimezone: true }),        // null = unread
    createdAt: timestamp("created_at", { withTimezone: true }).notNull().defaultNow(),
  },
  (table) => [index("guest_messages_invitation_id_idx").on(table.invitationId)],
);
export type GuestMessageRow = typeof guestMessages.$inferSelect;
```

- Photos as a `text[]` column (not a child table): a message is one unit with a
  handful of photos; array keeps create/read/delete single-row. Cascade-delete
  covers DB; S3 objects cleaned in the route.
- "At least one of message/photos" enforced in the route/zod, not the DB.
- Migration: `pnpm db:generate` → `0004_*.sql`; applied to RDS by the deploy
  pipeline (same path as `0003`).

### 3.2 S3 — new private `m/` prefix

`apps/api/src/s3.ts`:

- `createMessagePresignedUpload(invitationId, contentType, size)` → `{uploadUrl, key}`
  with `key = m/<invitationId>/<uuid>.<ext>` (clone of `createGuestPresignedUpload`,
  `s3.ts:85`).
- `createPresignedGetUrl(key, expiresIn = 600)` → presigned **GET** (new; uses
  `GetObjectCommand` + `getSignedUrl`). Used by the owner inbox to display
  private photos.
- `deleteAssetObjects(keys: string[])` → batch `DeleteObjectsCommand` (generalize
  the existing single-key `deleteAssetObject`).
- Extend `deleteAssetPrefix()` (`s3.ts:134`) to also `deletePrefix(\`m/${id}/\`)`
  so deleting an invitation removes its private message photos too.

**Bucket policy:** do **not** add `m/*` to the public-read statement
(`PublicReadUploads` currently covers `i/*, g/*`). `m/*` stays private; the
couple sees photos only through presigned GET. (Infra: no policy change needed
beyond confirming `m/*` is *not* public.)

### 3.3 Shared contracts — `packages/shared/src/index.ts`

```ts
// Private guest messages (QR inbox).
export const M_ASSET_KEY_REGEX =
  /^m\/[0-9a-f-]{36}\/[0-9a-f-]{36}\.(jpg|png|webp)$/;       // i/g style, m/ prefix
export const GUEST_MESSAGE_MAX_PHOTOS = 10;
// reuse GUEST_UPLOAD_CONTENT_TYPES + GUEST_UPLOAD_MAX_BYTES

export const guestMessagesSettingsSchema = z.object({         // → invitationDesignFieldsSchema.guestMessages
  enabled: z.boolean().optional(),
  openDate: z.string().datetime({ offset: true }).optional(), // optional open window
  prompt: z.string().optional(),                              // custom heading
});

export const createGuestMessagePresignInputSchema = z.object({
  contentType: z.enum(GUEST_UPLOAD_CONTENT_TYPES),
  size: z.number().int().positive().max(GUEST_UPLOAD_MAX_BYTES, "파일은 15MB 이하여야 합니다"),
});
export const createGuestMessagePresignResponseSchema = z.object({
  uploadUrl: z.string().url(),
  key: z.string(),                                            // m/<id>/<uuid>.<ext>
});

export const createGuestMessageInputSchema = z
  .object({
    senderName: z.string().trim().max(40).optional(),
    message: z.string().trim().max(1000).optional(),
    photoKeys: z.array(z.string().regex(M_ASSET_KEY_REGEX)).max(GUEST_MESSAGE_MAX_PHOTOS).default([]),
  })
  .refine((v) => Boolean(v.message) || v.photoKeys.length > 0, "메시지나 사진 중 하나는 필요합니다");

// Owner view (inbox): photos carry a short-lived presigned URL.
export const guestMessageSchema = z.object({
  id: z.string().uuid(),
  senderName: z.string().nullable(),
  message: z.string().nullable(),
  photos: z.array(z.object({ key: z.string(), url: z.string().url() })),
  readAt: z.string().datetime().nullable(),
  createdAt: z.string().datetime(),
});
export const guestMessageListResponseSchema = z.object({
  messages: z.array(guestMessageSchema),
  total: z.number().int(),
  unread: z.number().int(),
});
```

Add `guestMessages: guestMessagesSettingsSchema.optional().catch(undefined)` to
`invitationDesignFieldsSchema` (near `guestUpload`, `index.ts:400`).

### 3.4 API — new `apps/api/src/routes/guest-messages.ts` (register in `server.ts`)

| Method + path | Auth | Rate limit | Behavior |
|---|---|---|---|
| `POST /:id/messages/presign` | public | 30 / 10 min | published + `messageWindow` open + S3 configured; validates type/size; returns `{uploadUrl, key}` under `m/<id>/` |
| `POST /:id/messages` | public | 10 / 10 min | published + window open; validates body; each `photoKeys[i]` must `startsWith("m/<id>/")`; ≤ cap; inserts row; returns `201 {message:{id,createdAt}}` |
| `GET /:id/messages` | **owner** | — | lists newest-first; generates a presigned GET per photo key; returns `{messages, total, unread}` |
| `PATCH /:id/messages/:msgId/read` | **owner** | — | sets `readAt` (optional v1; enables unread badge) |
| `DELETE /:id/messages/:msgId` | **owner** | — | deletes row + best-effort `deleteAssetObjects(row.photoKeys)`; `204` |

- `messageWindow(invitation)` clones `uploadWindow()` against `guestMessages`.
- Gate-closed responses mirror `gateClosed()`:
  `guest_message_disabled` / `guest_message_not_open`.
- Owner helpers reused: `findPublishedInvitation`, `findOwnedInvitation`,
  `invalidInput`, `notFound`, `unauthenticated` (`apps/api/src/routes/http.ts`);
  `sessionUserId` from `apps/api/src/auth/auth.ts` (pattern: `guest-uploads.ts:134`).

### 3.5 Web — guest submit page

- **Route:** `apps/web/src/app/(invitation)/invitations/[slug]/send/page.tsx`
  (+ `send.module.scss`). Bare layout, `robots: noindex`.
- Server component resolves slug→published invitation (reuse `fetchPublished`);
  reads `guestMessages` from the design doc. If missing/disabled/not-published
  → a friendly "지금은 메시지를 받지 않습니다" notice (or `notFound`). If before
  `openDate` → a "곧 열립니다" notice.
- **Client island** `SendForm` (`"use client"`): name (optional), message
  textarea (optional), multi-photo picker (≤10, live thumbnails, remove).
  On submit, per photo: `reencodeToJpeg()` → `presignGuestMessageUpload()` →
  `PUT uploadUrl` → collect key; then `createGuestMessage({senderName, message,
  photoKeys})`; show a success screen ("마음이 전달되었습니다 💌"). Progress +
  error states modeled on `GuestUpload.tsx`.
- **lib:** add `presignGuestMessageUpload(id, {contentType,size})` and
  `createGuestMessage(id, body)` to `apps/web/src/lib/api.ts`.

## 4. Security & abuse model

- **Gating:** every guest endpoint requires `status === "published"` and the
  `guestMessages` window open (`enabled && (!openDate || now ≥ openDate)`).
- **Rate limits:** presign 30/10 min, create 10/10 min (per existing posture).
- **Upload integrity:** presigned PUT binds content-type + exact byte length
  cryptographically (existing `s3.ts` behavior); type enum + 15 MB cap; ≤10
  photos; EXIF stripped client-side via `reencodeToJpeg`.
- **Ownership:** `photoKeys` must sit under `m/<invitationId>/`; rejected
  otherwise (mirrors guest-upload confirm at `guest-uploads.ts:101`).
- **Privacy:** `m/*` is **not** public-readable; only the authenticated owner
  retrieves photos, via short-lived presigned GET. Keys are unguessable UUIDs.
- **XSS:** message/name rendered as text (React escapes); no HTML.
- **Known gap (accepted, matches 게스트스냅):** a guest who presigns+PUTs but never
  calls `POST /messages` leaves an orphan object under `m/`. Low volume; a
  future sweeper can prune `m/` objects not referenced by any row.

## 6. Open questions / for review

1. **Photo privacy — RESOLVED 2026-06-14: truly-private presigned GET.** The
   feature is a private couple inbox, so photos live under a non-public `m/`
   prefix and the owner views them via short-lived presigned GET; the public `g/`
   model was rejected (weaker privacy). Locked in §1 — Phase 1 unblocked.
2. **Couple notification** on a new message (email/push) — **out of scope** for
   v1; note as a fast-follow.
3. **QR on the invitation page too** (for online guests, not just print) —
   future; trivial once `/send` exists.
4. **Korean labels — CONFIRMED 2026-06-14.** Live wording: feature toggle
   "QR 메시지·사진 받기 (비공개)" (QR panel/title "QR 메시지·사진"); inbox
   "받은 메시지 (비공개)"; submit heading default

   The "(비공개)" qualifiers were added vs the original proposal to distinguish
   this private inbox from the public 방명록 / 게스트스냅.
