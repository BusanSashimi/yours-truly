### Editor timezone fix — `(site)/dashboard/invitations/[id]/page.tsx`
- Rewrote `isoToLocalInput`/`localInputToIso` to convert against a fixed
  **+09:00** (`KST_OFFSET_MS`) instead of the browser's timezone. Previously the
  ceremony time was round-tripped through the editor's local TZ while the guest
  page renders pinned to Asia/Seoul, so editing from abroad shifted the displayed
  hour (e.g. 13:00 typed in New York → guest page showed 새벽 2시). Now fully
  browser-TZ-independent; verified the exact logic under TZ=UTC / America/New_York
  / Asia/Seoul / Pacific/Kiritimati — all produce display `13:00` ⇄ store
  `2026-10-24T04:00:00.000Z` (matching the seed's documented 13:00 KST).
- Labeled the field `예식 일시 (한국 시간)` so the couple knows the time is Korean local.

  design to its seed shape (removed the template/heroImageKey I'd set for shots).
